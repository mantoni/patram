# v0 Conventions Proposal

- Kind: convention
- Status: active

## Config

- Config file: `.patram.json`.
- Single project config file in v0.
- Config format: JSON.
- Paths: repo-relative.
- Path separator: `/`.
- Include patterns: glob strings.
- Stored queries live beside schema config.
- Schema config includes `kinds`, `relations`, and `mappings`.
- Derived summaries live beside stored queries and schema config.
- Directive typing lives in `directive_types`.
- Reusable repo path policies live in `path_classes`.
- Per-kind directive rules live in `metadata_schemas`.

```json
{
  "include": ["docs/**/*.md"],
  "queries": {
    "active-plans": {
      "where": "kind=plan and status=active"
    },
    "decision-review-queue": {
      "where": "kind=decision and status=proposed"
    },
    "ready-tasks": {
      "where": "kind=task and status=ready"
    }
  },
  "derived_summaries": {
    "plan_execution": {
      "kinds": ["plan"],
      "fields": [
        {
          "name": "execution",
          "select": [
            {
              "when": "count(in:tracked_in, kind=task) > 0 and none(in:tracked_in, kind=task and status not in [done, dropped, superseded])",
              "value": "done"
            }
          ],
          "default": "not_started"
        },
        {
          "name": "total_tasks",
          "count": {
            "traversal": "in:tracked_in",
            "where": "kind=task"
          }
        }
      ]
    }
  },
  "directive_types": {
    "status": {
      "type": "string"
    },
    "tracked_in": {
      "type": "path",
      "path_class": "plan_docs"
    }
  },
  "path_classes": {
    "plan_docs": {
      "prefixes": ["docs/plans/"]
    },
    "task_docs": {
      "prefixes": ["docs/tasks/"]
    }
  },
  "metadata_schemas": {
    "task": {
      "document_path_class": "task_docs",
      "unknown_directives": "error",
      "directives": {
        "status": {
          "presence": "required",
          "type": {
            "type": "enum",
            "values": ["pending", "ready", "in_progress", "blocked"]
          }
        },
        "tracked_in": {
          "presence": "required"
        },
        "execution": {
          "presence": "forbidden"
        }
      }
    }
  },
  "kinds": {
    "document": {
      "builtin": true
    }
  },
  "relations": {
    "links_to": {
      "builtin": true,
      "from": ["document"],
      "to": ["document"]
    }
  },
  "mappings": {
    "document.title": {
      "node": {
        "kind": "document",
        "field": "title"
      }
    },
    "markdown.link": {
      "emit": {
        "relation": "links_to",
        "target_kind": "document",
        "target": "path"
      }
    }
  }
}
```

## Parsing

- Markdown first.
- Title: first line.
- Heading title: strip leading `#`.
- Link claim type: `markdown.link`.
- Directive claim type: `directive`.
- Directive names normalize to `lower_snake_case`.

## Identity

- Document node id: `doc:<path>`.
- Non-document node id: `<kind>:<key>`.
- Claim id: `claim:<document_id>:<ordinal>`.

## Graph

- Built-in kind: `document`.
- Built-in relation: `links_to`.
- v0 graph layers: `schema`, `claims`, `graph`.
- Graph nodes are concrete objects.
- Kinds and relations live in `.patram.json`.
- Query target: graph nodes.

## CLI

- Root command: `patram`.
- Config discovery: current working directory first.
- Default target for `check`: current working directory.
- `query --where` runs an ad hoc query.
- `query <name>` runs a stored query.
- `queries` lists stored queries.
- `show` requires one file path.

## Output

- Human-readable text first.
- Canonical `plain`, `rich`, and `json` output layouts live in
  `docs/conventions/cli-output-v0.md`.
- `check` uses the shared diagnostic report contract for config and content
  validation.
- Missing required directives and invalid placement report at `1:1`.
- Directive value failures report the directive origin.

## Query

- Query command name: `query`.
- Stored queries live in `.patram.json`.
- Stored query lookup is exact by name.
- v0 query language is small.
- v0 query language filters nodes only.
- v0 query language supports exact relation-target terms.
- v0 query language supports one-hop traversal aggregates.
- v0 query language supports set-membership terms.

```txt
kind=task and status in [pending, ready, in_progress, blocked]
tracked_in=doc:docs/plans/v0/worktracking-agent-guidance.md
kind=decision and status=accepted
implements_command=command:query
status not in [done, dropped, superseded]
none(in:tracked_in, kind=decision)
path^=docs/roadmap/
title~automation
```
