# v0 Conventions Proposal

## Config

- Config file: `.patram.json`.
- Config format: JSON.
- Paths: repo-relative.
- Path separator: `/`.
- Include patterns: glob strings.

```json
{
  "include": ["docs/**/*.md"]
}
```

## Parsing

- Markdown first.
- Title: first line.
- Heading title: strip leading `#`.
- Link claim type: `markdown.link`.
- Directive claim type: `directive`.
- Directive names normalize to `lower_snake_case`.

## Identity

- Document node id: `doc:<path>`.
- Non-document node id: `<kind>:<key>`.
- Claim id: `claim:<document_id>:<ordinal>`.

## Graph

- Built-in kind: `document`.
- Built-in relation: `links_to`.
- v0 graph layers: `schema`, `claims`, `graph`.
- Graph nodes are concrete objects.
- Kinds and relations live in schema.

## CLI

- Root command: `patram`.
- Config discovery: current working directory first.
- Default target for `check`: current working directory.
- `show` requires one file path.

## Output

- Human-readable text first.
- No JSON CLI output in v0.
- Diagnostic format: `<path>:<line>:<column> <level> <code> <message>`.
- `show` output order:
  - original source
  - resolved summary

## TDD
